# 서비스 흐름 및 프롬프트 정책

## 1. 핵심 서비스 루프

입력 → 진단 → 플랜 → 문제 생성 → 풀이 → 오답 저장 → 반복 출제

## 2. 기능 흐름 상세

1) 사용자 입력
- 시험명, 과목별 점수, 목표일, 하루 학습 시간

2) 약점 진단
- 40점 이하 과목 우선
- 오답 반복 횟수 반영

3) 플랜 생성
- 하루 3~5개 task
- daily_time 초과 금지

4) 문제 생성
- 과목/난이도/개수 기반 GPT 호출

5) 결과 반영
- 정오답 기록
- 오답 큐 스케줄링

## 3. 프롬프트 정책

### 시스템 프롬프트 원칙
- 불필요한 설명 금지
- 실행 가능한 계획 제시
- 오답 반복 구조 강제
- 짧고 실전형 표현

### 문제 생성 프롬프트
- 객관식 4지선다
- 정답/해설 포함
- 최신 기출 스타일 반영

### 오답 재출제 프롬프트
- 동일 개념, 다른 질문 방식
- 난이도 동일 또는 약간 하향

## 4. 출력 계약

- 오늘 할 것 (3~5개)
- 예상 문제 (10~20개)
- 정답 + 핵심 해설
- 약점 분석
- 합격 확률
