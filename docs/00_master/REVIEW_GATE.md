# 개발 착수 승인 게이트 (Review Gate)

본 문서는 **문서 승인 완료 전 코드 구현 금지** 원칙을 운영하기 위한 체크리스트입니다.

## Gate-0. 비전/범위 승인

- [ ] 제품 목표(합격 지원) 승인
- [ ] 1.0 범위(Must/Won't) 승인
- [ ] 지원 시험 범위 승인(정보처리기사 필수, TOEIC 여부)

## Gate-1. 기술 방향 승인

- [ ] 프론트 스택 확정 (현행 Vite 유지 여부)
- [ ] 백엔드 언어 확정 (FastAPI/NestJS)
- [ ] 인증 방식 확정 (Supabase/Clerk/Firebase)
- [ ] 배포 방식 확정 (Vercel + Railway 등)

## Gate-2. 데이터/API 승인

- [ ] 핵심 엔티티(users, subject_scores, wrong_answers 등) 승인
- [ ] 필수 API(`GET /study/today`, `POST /study/submit`) 승인
- [ ] GPT 응답 JSON 스키마 승인

## Gate-3. 실행 계획 승인

- [ ] 프론트 F0~F3 단계 승인
- [ ] 백엔드 B0~B4 단계 승인
- [ ] 일정(8~10주) 승인
- [ ] 리스크 대응 전략 승인

## 최종 착수 조건

아래 모두 충족 시 구현 착수 가능:

- [ ] 위 Gate-0~3 모두 체크 완료
- [ ] 본 문서 상태를 Approved로 변경
- [ ] 착수 브랜치 전략 명시
