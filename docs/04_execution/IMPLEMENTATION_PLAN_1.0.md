# 구현 계획서 (Implementation Plan 1.0)

## 1. 실행 원칙

- 개발 착수 전 문서 승인 완료
- 단계별 범위 고정 후 구현
- 각 단계 종료 시 데모/검토 수행

## 2. 단계별 계획

### Phase F0 (프론트 기반 정리, 1주)

- 전역 상태 구조 정리 (`selectedCert`, `scores`, `todayPlan`)
- API 레이어/에러 처리 공통화
- 환경 변수 및 실행 스크립트 정리

완료 기준:
- 페이지 간 자격증 상태 일관성 확보
- mock 데이터 의존 지점 식별 완료

### Phase F1 (온보딩/설정 연동, 1주)

- 온보딩 플로우(시험/점수/시험일/학습시간)
- 온보딩 미완료 사용자 가드
- 설정값 저장 연동

완료 기준:
- 신규 사용자 입력값이 저장/재조회 가능

### Phase F2 (대시보드/학습 플랜 연동, 1주)

- 대시보드의 고정 수치 제거
- `GET /study/today`, `GET /analytics/dashboard` 연동
- 학습 플랜 시작 동선 연결

완료 기준:
- 오늘 할 일/합격확률/약점 데이터 실연동

### Phase F3 (문제풀이/오답 루프, 1.5주)

- 문제 생성 API 연동 (`POST /ai/generate-quiz`)
- 풀이 제출/오답 저장 (`POST /study/submit`)
- 오답노트 라우트 및 재출제 기능

완료 기준:
- 생성 -> 풀이 -> 오답저장 -> 복습 재출제 루프 동작

## 3. 백엔드 동시 계획

### B0 (셋업, 3일)
- 프로젝트 초기화, DB 마이그레이션 기반 구성

### B1 (Auth/User, 4일)
- 인증/유저/자격증/점수 저장

### B2 (Study Engine, 5일)
- 오늘 플랜 생성/조회, 복습 큐 조회

### B3 (GPT Orchestrator, 5일)
- 문제 생성/오답 재출제 및 JSON 파싱 안정화

### B4 (Analytics, 3일)
- 합격확률/추이/집중도 API

## 4. 통합 마일스톤

- M1: 온보딩 + 기본 데이터 저장
- M2: 대시보드 실데이터 반영
- M3: 오답 반복 루프 완성
- M4: 1.0 배포 후보
